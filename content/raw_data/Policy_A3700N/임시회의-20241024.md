# 임시회의-20241024

## 안건 1: Pickup Event 저장 여부 관련 설정 추가

### 참고 — 구현 상황 / 매뉴얼

> [이미지 image 104.png] [참고] 김용운 이메일(2024-10-04) "Pickup 이벤트 저장 기능 구현 상황 공유". Pickup 이벤트 저장 여부: A2500=X, **A3700N=O(설정으로 결정, 디폴트 Disable)**. 저장 활성화돼도 Delay Time이 0이면 Pickup 이벤트 미발생.

> [이미지 image 105.png] [참고] 조국춘 이메일(2024-10-07) A3700 사용자 매뉴얼 발췌. Pickup/Start/End 이벤트는 동작상 항상 발생하나 **로그 기록 여부가 설정 대상**(디폴트 비기록). 설정 항목 Setup > System > Pickup Event Log(0:Disable/1:Enable, 디폴트0). Event Detection(Over/Under/Off, Threshold/Time delay/Hysteresis) 설명 포함.

> [이미지 image 106.png] [참고] Accura 3700 Communication Guide(Rev2_30) - Event Save Mode Setup. Register 5501(access for event save mode setup), **5502 Save pickup event(0:Do Not Save / 1:Save, 디폴트0)**.

> [이미지 image 107.png] [참고] 동일 Communication Guide의 5502 "Save pickup event" 행 확대(UInt16, 디폴트 0).

- Asis: Pickup Event 저장 설정이 있으나 **장치 화면에는 미표시**(Modbus Poll로만 설정, AccuraSM에도 항목 없음).

### Tobe — "Pickup Event Log" 항목 추가 (AccuraSM에도 추가). 범위 0:Disable(디폴트)/1:Enable

> [이미지 image 108.png] [Tobe/1안] SETUP > SYSTEM > Locale에 **"Pickup Event Log = Disable" 항목 추가**(빨강). DI/DO Counter는 "Not Displayed".

> [이미지 image 109.png] [Tobe/2안] SETUP > EVENT > **Etc. 부메뉴 신규**, Pickup Event Log = Disable.

> [이미지 image 110.png] [Tobe/2안] SETUP > EVENT > **Advanced 부메뉴 신규**, Pickup Event Log = Disable.

**👉 결정: Tobe(1안, Locale에 추가)으로 결정.** 설정항목 순서 수정(Temperature Unit/Timezone, Measurement/Energy Unit, Comm. Event Log/Pickup Event Log, DI·DO Closed Status Color/DI·DO Counter). A2500에도 추가 필요.

## 안건 2: 「SETUP > NETWORK > Ethernet」 DHCP 설정 시 화면 표시 수정

> [이미지 image 111.png] [참고] A3700N Modbus Map. DHCP(3722, 0:Disable/1:Enable), **DHCP 서버에서 받은 동적 정보** IP address(3723)·Subnet mask(3725, Range 8~30 = 255.0.0.0~255.255.255.252)·Gateway(3726) 레지스터(노랑).

### Asis (DHCP 무관하게 Static만 표시)

> [이미지 image 112.png] [Asis] SETUP > NETWORK > Ethernet, **DHCP Disable**. IP 10.10.10.100 / Subnet 255.255.255.0 / Gateway 10.10.10.1 (Static).

> [이미지 image 113.png] [Asis] 동일 화면, **DHCP Enable여도 Static 정보 그대로 표시**(동적 정보는 통신맵에만 존재, 화면 미표시).

### Tobe (DHCP에 따라 Static/Dynamic 구분 표시)

- DHCP Disable: Static 표시(Asis와 동일). DHCP Enable: Dynamic 표시 + ReadOnly.

> [이미지 image 114.png] [참고] DHCP 서버 장애 시 PC의 APIPA(Automatic Private IP Addressing) 주소 할당 설명 — 169.254.0.1~169.254.255.254 범위.

> [이미지 image 115.png] [참고] APIPA 시 Subnet Mask는 항상 255.255.0.0, Gateway는 0.0.0.0으로 설정됨(로컬 외부 통신 불가).

> [이미지 image 116.png] [참고] IP 0.0.0.0의 의미("지정되지 않은 주소"/"모든 주소") — 일반 통신용 고정 IP로는 사용 불가.

> [이미지 image 117.png] [Tobe] DHCP Enable 시 **Dynamic 정보를 ReadOnly로 표시**(IP 10.10.26.142 / Subnet 255.255.224.0 / Gateway 10.10.26.1, 회색 비활성).

> [이미지 image 118.png] [Tobe] **DHCP 정상 동작 안 할 때**(통신연결 문제 등): IP·Gateway "0.0.0.0", Subnet "255.255.0.0"으로 ReadOnly 표시.

**DHCP 설정변경 Entered(Not Applied) 시 Highlight 규칙** — Highlight는 설정조회화면 대비 값이 변경된 항목에만 적용. (Disable→Enable: 0.0.0.0&255.255.0.0, ReadOnly. Enable→Disable: Static값, Normal)

> [이미지 image 119.png] [예시] 설정조회화면(DHCP Disable, Static 10.10.10.100/255.255.255.0/10.10.10.1, Normal). DHCP Enable로 변경했다 다시 Disable로 되돌리면 이 화면과 동일.

> [이미지 image 120.png] [예시] DHCP Enable Entered(Not Applied). DHCP·IP·Subnet·Gateway 모두 0.0.0.0(&255.255.0.0)로 변경, **ReadOnly + Highlight(초록)**, Apply/Cancel 버튼.

> [이미지 image 121.png] [예시] 설정조회화면(Subnet Mask가 이미 255.255.0.0인 경우).

> [이미지 image 122.png] [예시] DHCP Enable Entered. IP·Gateway는 Highlight되나 **Subnet Mask(255.255.0.0)는 값 변경이 없어 Highlight 안 됨**.

> [이미지 image 123.png] [예시] DHCP Disable Entered(Not Applied). 기존 Enable→Disable 전환 시 Static값(10.10.10.100 등) ReadOnly + Highlight(초록).

> [이미지 image 124.png] [예시] 설정조회화면(DHCP Enable, Gateway 10.10.10.1).

> [이미지 image 125.png] [예시] DHCP Disable Entered. IP·Subnet은 Highlight되나 **Gateway(10.10.10.1)는 값 변경 없어 Highlight 안 됨**.

**👉 결정: Tobe 안으로 결정.** A2500은 추후 요청 시 추가.
