# 임시회의-20230706

## 안건: 「SETUP > SYSTEM > Locale > Measurement Unit」 "V/A" 설정 시 화면 표시 정책 보완

- 문제: Measurement Unit을 "V/A"로 설정 시, 표시값이 7자리 이상이면 잘림 현상 발생(PT비 최대 설정 시). PNT팀 "A3700Nv1.6.8p7 Release 사전 Test"에서 발견.

### Asis (잘림 현상)

> [이미지 Untitled 28.png] [Asis] 실제 장치(Accura 3700) HOME Dashboard, **Auto unit change** 모드. VOLTAGE 262.0 MV / CURRENT 298.8 kA / POWER 129.1 TW — 단위 자동변환으로 값이 잘리지 않음.

> [이미지 Untitled 29.png] [Asis] HOME Dashboard, **V/A** 설정. VOLTAGE가 "2065312 V"처럼 7자리 이상이 되어 **앞자리 잘림 현상 발생**(CURRENT 298846 A). Dashboard 잘림 O.

> [이미지 Untitled 30.png] [Asis] HOME > Voltage(RMS, L-L), Auto unit change. AB 217.8 MV / BC 304.1 MV / CA 264.7 MV / Average 262.2 MV — 잘리지 않음.

> [이미지 Untitled 31.png] [Asis] HOME > Voltage(RMS, L-L), **V/A** 설정. AB 7803040 V / BC 4223360 V / CA 4719136 V / Average 2248528 V — 7자리 이상이라 **잘림 현상 발생**. Voltage 잘림 O.

> [이미지 Untitled 32.png] [Asis] HOME > Current(RMS), Auto unit change. A 149.0 kA / B 299.1 kA / C 448.0 kA — 잘리지 않음.

> [이미지 Untitled 33.png] [Asis] HOME > Current(RMS), **V/A** 설정. A 149025 A / B 299003 A / C 448191 A / Average 298740 A — 6자리로 **잘림 현상 없음**. Current 잘림 X.

### Tobe (개선안)

- 정책: "V/A" 설정에서 표시값이 7자(1000단위 구분기호 "," 포함) 초과 시, "Auto unit change"와 동일하게 단위 표시로 동작. 단, 설정값은 "V/A" 유지.
