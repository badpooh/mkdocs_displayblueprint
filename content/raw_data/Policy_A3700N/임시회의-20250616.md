# 임시회의-20250616

## 안건 [01]: 「HOME > SYSTEM」 Logic 상태 정보 표시 추가

- 배경: A3700N-Logic 연동(CustomEvent 기능의 로직 연동). KT P&M 요구로 진행.

### 참고 — 요구사항/논의 이메일 (노수장 차장)

> [이미지 image.png] 노수장 차장 이메일(2025-06-11). 제목 "A3700N-Logic 연동 관련 건(CustomEvent 기능의 로직 연동)". 요구사항: 현장 타사 Protection Relay(DVR) 오동작 빈번 → Accura 3700N 계측데이터로 유사 기능 구현, Custom Event 활용. 제약: 차단기 제어는 Main이 아닌 Logic Application이 담당, DO 모듈 부족으로 Custom Event 연동 DO 출력 불가.

> [이미지 image 1.png] 회신 이메일(2025-06-12), 6/11 회의 논의 정리. ①Custom Event-Logic 연동(80채널 중 최대 16채널, 통신맵에 채널당 Start 상태 1bit 할당) ②Logic 상태정보 UI(로직 동작 시에만 표시, 통신맵 65535 page 제공) ③Ethernet LED 제어(로컬 Modbus로 LED 점멸→차단 검토) ④Logic 포함 양산방식(F/W patch number만 증가, 납품 전 로직파일 삭제 기능) ⑤테스트.

### 참고 — A2750LM 기존 화면

> [이미지 image 2.png] [참고/A2750LM] HOME > SYSTEM > Logic Status Information 화면. Logic Status(Disable), Logic Version(0.0.000), Logic Period[msec](0) 표시. 상태 종류: Disable/No Logic/Error/Running/Stopped.

### Asis (A3700N 기존)

> [이미지 image 3.png] [Asis] A3700N HOME > SYSTEM > Description 화면. 좌측 부메뉴 Description/Local Time/A3700/Network State/Temperature만 존재하고 **Logic 부메뉴 없음**(빨강 빈 영역).

### Tobe (개선안)

- 「Logic」 부메뉴 추가 (로직이 동작 중인 경우에만 표시). 표시 항목: 로직 동작 상태, 로직 어플리케이션 버전, 실행 중인 로직 버전.

> [이미지 image 4.png] [Tobe] A3700N HOME > SYSTEM > **Logic Information 부메뉴 추가**. Logic State(Running), Application Version(0.00.0), User Logic Version(0.0.000) 표시.

### 결정사항 👉 (Tobe 안으로 결정)

- "Logic Information" 부메뉴 추가. 표시: Application Version, User Logic Version, State(용어는 "Status" 대신 "State"로 통일).
- A2750LM과 달리 A3700N은 어플리케이션 버전 + 유저 로직 버전 모두 표시 ("유저" 접두사로 구분).
- **로직 하트비트가 들어와 "Running" 상태일 때만 부메뉴 표시**, 그 외 상태(Disable/No Logic/Error 등)는 부메뉴 자체 미표시(단순화).
- 설정으로 표시 여부를 조정하는 대안은 셋업매니저/에디터 개입이 번거로워 미채택.
- 향후: 에디터로 로직 다운로드 시 운영 형태 전환, 기존 로직 삭제(디지털) 기능 구현 예정.
