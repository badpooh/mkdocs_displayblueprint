    요약

    이번 회의에서는 CMS2P 모듈의 페어링 기능과 관련된 UI 표시 방식에 대해 심도 있게 논의했습니다. 주요 논점은 메인 모듈과 서브 모듈 간의 페어링 동작 방식, 화면 표시 방법, 그리고 비정상 상태 처리에 관한 것이었습니다.

    ### 페어링 메인 모듈 기능 정의

    - 현재 "페어링 메인"이라는 용어가 데이터 가공 측면에서 오해의 소지가 있음
    - 메인 모듈은 대표 ID로서의 역할만 하며, 데이터 취합/공개는 하지만 데이터 가공을 담당하는 것은 아님
    - 페어링 관련 설명 문구를 수정하여 정확한 개념을 전달할 필요가 있음

    ### 페어링 동작 원칙

    - 메인 모듈이 우선권을 가지며, 서브 모듈은 메인 모듈의 설정값을 따름
    - 용량이 다른 모듈 간 페어링 시(예: 250A와 60A) 메인 모듈의 설정을 기준으로 동작
    - 페어링 해제 시 서브 모듈은 자신의 정격에 맞는 값으로 클리핑 처리됨 (예: 60A 초과 값은 60A로 제한)
    - 레이티드 커런트 설정은 부하율 계산에 영향을 미침

    ### UI 표시 방식

    - 표시 상태를 정상/기본/비정상 세 가지로 구분
    - 인접한 모듈만 페어링을 허용하고 정상으로 표시
    - 인접 모듈은 체인 뷰에서 네모 박스로 묶어서 표시
    - 비인접 모듈 페어링 시 경고 아이콘 표시로 비정상 상태 알림
    - 메인 모듈에 우선권을 부여하여 화면 표시(예: 온도 옵션이 있는 경우)
    - 상단 바에는 메인 모듈의 ID 색상 표시, 서브 모듈은 ID와 3P4W 설정값 표시

    ### 비정상 상태 처리

    - 자기 자신의 ID로 서브 모듈 설정 시 비정상 표시
    - CMS-2P가 아닌 다른 모듈을 서브로 지정 시 비정상 표시
    - 이미 페어링된 모듈을 다른 모듈의 서브로 지정 시 비정상 표시
    - 비정상 상태는 2초간 토글 형식으로 "페어링 미스매치" 문구 표시
    - 비정상 상태도 화면에 표시하여 사용자가 인지할 수 있도록 함

    ### 페어링 설정 방식

    - 메인 모듈에서만 페어링 설정 가능
    - 모드를 3P4W로 변경 시 페어링 모듈 ID 설정 항목이 활성화됨
    - 페어링 모듈 ID의 기본값은 1로 설정
    - 설정 적용 시 선택한 ID가 화면에 표시됨

    요약

    회의에서는 네트워크 장비 포트의 LED 표시 동작과 관련된 사용자 경험 문제를 논의했습니다.

    ### 현재 LED 동작 상태

    - 이더넷 포트:
        - 주황색 LED: 연결 시 점등, 미연결 시 소등
        - 초록색 LED: 연결 시 점등, 통신 중일 때 점멸, 미연결 시 소등
    - RJ45 포트:
        - 우측 초록색 LED(TX)가 "노링" 모드에서는 연결 상태와 무관하게 계속 점멸
        - "링" 모드에서는 포트에 따라 다르게 동작 (1번 포트만 계속 점멸)

    ### 사용자 경험 문제점

    - 윤종훈 이사가 문제를 제기: 연결되지 않은 상태에서도 LED가 계속 점멸하는 것이 사용자에게 혼란을 줄 수 있음
    - 개발자들은 이 동작에 익숙해져 있어 문제를 인식하지 못했음
    - 사용자 관점에서 동일한 형태의 포트가 서로 다른 표시 방식을 사용하는 것도 혼란 요소

    ### 모델별 표시 방식 차이

    - 2700 모델: 실크로 TX, RX 및 "Speed", "Link" 표시가 명확히 되어 있음
    - 2500/LCG 모델: 실크 표시 없이 포트만 존재

    ### 논의된 대안

    - 현재 방식 유지
    - TX 송신이 없을 때 소등되는 조건 추가

    ### 결론

    회의에서는 명확한 결정이 내려지지 않았으나, 현재 방식을 유지하는 쪽으로 기울어진 것으로 보임. 추가 검토가 필요할 수 있음.
