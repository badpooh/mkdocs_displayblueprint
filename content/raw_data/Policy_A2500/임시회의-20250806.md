# 임시회의-20250806

## 안건: [01] [CMS-2P] 3P4W 결선모드 화면설계 완료 공유

- 「임시회의-20250403」·「20250529」·「20250612」·「20250710」·「20250722」 재논의 안건, 「임시회의-20250819」 재논의·「임시회의-20250930」에서 결정됨. 직전 결정(정격이 다른 경우도 비정상 표시, 인접 시 테두리·비인접 시 각각 링크정상아이콘)을 반영한 화면설계 완료 공유.
- Pairing 동작: Pairing Main(데이터 송수신 주체, 대표 ID 역할)/Sub, 설정은 Pairing Main에서만(Pairing Module ID 설정범위 0~39·디폴트는 구현 편의상 2 검토). 정상/기본/비정상 3가지 표시 상태. 비정상 표시 조건: 정격 상이·Main CM을 Sub로 설정·자신 ID를 Pairing Module ID로 설정·Sub가 CMS-2P 아님·Sub 문제 상황(ID Conflict/탈락)·이미 Pairing된 CMS-2P를 다른 모듈에서 Pairing 설정.

> [이미지 image.png] [Tobe/Chain View 인접 테두리, HOME] HOME > Chain View — 00 CMS·01 CMS가 보라색 테두리로 묶여 표시(Pairing Main/Sub 인접 시 테두리, 정상/기본 표시), 02 CMS·03 CMZ·04 DCM·05 TEMP·06 GW. (비인접 시 각각 링크정상아이콘, 비정상 시 Pairing Main에만 링크오류아이콘 — 「임시회의-20250710」 참고.)

**👉 화면설계 완료 공유** — 기본 표시는 Main 온도옵션 있고 Sub 온도옵션 없는 경우(온도 화면에서 Main 온도 2개 표시·Sub 온도 2개 하이픈 처리). 설정화면 UI(Pairing Module ID 디폴트 1 vs 2, Standby ID 팝업 형태, 메인 CM ID 영어 표기 영업 요청 확인)는 추후 재검토. Pairing Module ID 디폴트는 영업팀에 고객 요청사항 확인 필요(개인 의견이 아닌 고객 요청 기반으로 결정).
