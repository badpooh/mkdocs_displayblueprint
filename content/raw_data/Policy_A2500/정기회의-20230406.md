# 정기회의-20230406

## 안건: 괄호 "(" 앞문자 붙여쓰기/띄어쓰기 정책 재논의

- 「정기회의-20230202」에서 괄호 "(" 앞문자 **붙여쓰기**로 결정했으나, 영문에서는 띄어쓰기가 맞다는 문제 제기(2023/03/30). 대상: 「HOME > CURRENT > Leakage > 타이틀」, 모듈선택창.
- 1안: 앞 글자가 영문/숫자면 띄어쓰기, 국문이면 붙여쓰기(예: 국문(국문) / Korean (국문)). 어문규범상 "Korean(국문)"이 맞으나 루텍은 규칙 단순화·가독성 위해 "Korean (국문)"으로 띄어쓰기.
- 2안: 모두 띄어쓰기(불가피 시 붙여쓰기 허용). 어문규칙보다 가독성 중시.

**👉 Tobe "2안"(모두 띄어쓰기)으로 결정.** 재수정 필요 — 「HOME > CURRENT > Leakage > 타이틀」: "Leakage Current (ZCT)", 모듈선택창: "ID 01 (CM-3P-60A)".

## 안건: 「SETUP > SYSTEM > LED > Event LED Hold Time [sec]」 "3-in-1" 팝업 (Disable·숫자범위·Infinity)

- 부제: 1개 설정항목이 3개 설정(Disable, 숫자범위, Infinity)을 포함하면 **"3-in-1" 팝업**으로 표시. A7300 회의에서 논의됐으나 A7300엔 Event LED가 없어 A2500·A3700에 적용. (「정기회의-20230511」 최종결정, 「정기회의-20230420」 관련안건.)

## 안건: 「SETUP > … > 숫자패드형 수정팝업」 범위 표시 기호 및 숫자패드 형태 통일

- 범위 표시 기호: Asis는 문서엔 "–", 장치엔 "~". 물결표 "~"는 전기분야서 교류를 뜻해 UL이 범위표시에 사용 자제 요청. 1안 "-"(하이픈), 2안 "–"(en dash), 3안 "—"(em dash). (「정기회의-20230420」 최종결정.)

> [이미지 Untitled 18.png] [참고/기호입력] Windows 기호창(Ω 탭) — en dash "–" 등 선택(윈도우키 + ".").

> [이미지 Untitled 19.png] [참고/MS-Word 200%] 3안 비교 — 1안 "-"(hyphen), 2안 "–"(en dash), 3안 "—"(em dash) / "-20 - 70℃", "-20 – 70℃", "-20 — 70℃".

> [이미지 Untitled 20.png] [참고/MS-Word 100%] 동일 3안 비교(100% 보기 — 길이 차이 작음).

> [이미지 Untitled 21.png] [참고/장치 Roboto 16pt Bold] 동일 3안 비교(장치 폰트에서의 표시).

> [이미지 Untitled 22.png] [Tobe/결정 기호] 기호창 — en dash "–" 1개 선택(2안 결정).

- Tobe: **"–"(en dash, 2안)**. 대외용 문서(장치화면·매뉴얼·홈페이지)는 "–" 기본, 대내용 문서(노션·스마트시트·설계문서)는 "~" 허용.
- 숫자패드 형태 통일: A7300 회의(「정기회의-20220517」)에서 결정 — "Type 1" 기본, 상단 선택항목(En/Dis) 있으면 "Type 2" 선택적 사용, "Type 3"는 Password 전용.

> [이미지 Untitled 23.png] [Tobe/숫자패드 3종] Type 1(3열, 우측 X·Clear), Type 2(4열, 상단선택용), Type 3(Password, 2행 1~0). "±" 버튼은 "-"로 수정.

**👉 범위기호 "2안(en dash –)"으로 결정.** 숫자패드는 A7300 회의 디자인시안 따름, "±" 버튼을 "-"로 수정.
