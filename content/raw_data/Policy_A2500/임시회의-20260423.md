## 안건: [06] [A2550CMS-2P] 외부ZCT 인식 정책 개선

- CMS-2P ZCT 인식 정책 개선(2026-04-10 김용운·문부기). (A2550CMS-2P_ZCT_인식_기능(260417).docx 별도 첨부.)

> [이미지 image 13.png] [참고/메일 김용운 2026-04-10] "CMS-2P ZCT 인식 정책 개선 방향 논의" — PNT 테스트 중 CMS-2P Leakage 이벤트 감지 불능 현상. CMS-2P ZCT를 Fd.1만 연결 후 ZCT 인식 수행 시 Fd.1만 validity 생성·Fd.2 validity 없음, 이후 Fd.2 ZCT 연결해도 ZCT validity 누락으로 이벤트 미판단. CPU는 ZCT enable 설정으로 인식 화면 제공(A2500D만 활용), CMS-2P는 ZCT 인식 수행 안 해도 Leakage 계측 수행. 개선: CMS-2P에 ZCT enable 설정 항목 추가·비휘발 저장, enable 활성화(최초 1회) or 모듈 연결 시 자동 ZCT 인식 수행하여 validity 항상 유효, 기존 수동 ZCT 인식 유지.

> [이미지 image 14.png] [참고/메일 문부기 2026-04-10] "FW: CMS-2P ZCT 인식 정책 개선 방향 논의" — CMS-2P ZCT 인식 기능 초기 설계와 다르게 구현된 부분 검토. 상위 측에서 ZCT enable 지정 시점 또는 reboot 시점에 CMS-2P가 스스로 인식기능 동작 후 validity 값으로 상위 전달하는 안.

> [이미지 image 15.png] [참고/메인 SETUP DEVICE External ZCT Check 설정, MAIN] SETUP > DEVICE > External ZCT — "All External ZCT Connections Check"(ZCT Check 설정).

> [이미지 image 16.png] [참고/All External ZCT Connections Check 결과 팝업, MAIN] "All External ZCT Connections Check" 결과 — 01 Connected·02 "ZCT1 Disconnected, ZCT2 Connected"·03 Connected·04 Disconnected(ZCT Check 결과 팝업).

> [이미지 image 17.png] [참고/분기 SETUP Leakage 외부ZCT Enable, ID01 CMS-2P-60A+Z] SETUP > MEASUREMENT > Leakage — External ZCT Connection Enable(빨간 박스).

- Asis: CMS-2P 외부ZCT 연결 상태에서 ZCT Check 설정 실행해야만 ZCT Validity 생성·이벤트 판단(Validity "0"이면 누설전류 이벤트 미판단).
- Tobe: ZCT Check 미실행해도 ① External ZCT Connection Enable(최초 1회)·② CMS-2P 외부ZCT 모듈 재부팅 시 ZCT Validity 생성(결과 팝업 미표시). 기존 ZCT Check 기능은 사용자 필요 시 직접 수행 유지.

**👉 Tobe 안으로 결정** — (선 반영 완료, 기획적 점검 후 2500 메인에 함유하여 일괄 점검 예정.)
