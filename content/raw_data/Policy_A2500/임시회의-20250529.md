# 임시회의-20250529

## 안건: [01] [CMS-2P] 「분기 > SETUP > MEASUREMENT > Load Mode」 Mode 설정 범위 수정

- CMS-2P 결선모드 배치방식 조정 요청에 따른 Load Mode 설정 범위·순서·디폴트 수정.

> [이미지 image.png] [참고/메일 김민수 2025-05-08] "RE: CMS-2P 결선모드 배치방식 조정 요청" — "1P2W" 이름은 기존 유지하기로 금일 오전 회의에서 결정. 1P2W일 때 온도는 2개 표시·전류는 1개 상만 표시하는 기존 설계 유지(차후 현장 요구 발생 시 재검토).

> [이미지 image 1.png] [참고/메일 문부기 2025-05-08] "CMS-2P 결선모드 배치방식 조정 요청" — wiring 모드 배치를 전압 결선모드 순서와 유사하게 조정·통신맵 대응값도 조정. 0:3P4W·1:3P3W(기본값)·2:1P2W·3:1P2F. 작업지점 CMS-2P·A2500M/D. 3P4W는 미공개이므로 0번은 reserved 처리·화면 미표시. PNT팀 8차테스트 5/13, 결선 변경작업 5/14 목표.

> [이미지 image 2.png] [참고/분기 MEASUREMENT Load Mode, ID01 CMS-2P-60A] SETUP > MEASUREMENT > Load Mode — Mode 1P2F. 항목범위 0:1P2F·1:1P2W·2:3P3W, 디폴트 0:1P2F(CM-3P·CMM-3P와 범위·디폴트 다름).

- Asis: 0:1P2F(디폴트)·1:1P2W·2:3P3W·3:3P4W.
- Tobe: 0:3P4W·1:3P3W(디폴트)·2:1P2W·3:1P2F.

**👉 Tobe 안으로 결정** — 위 안건들은 공유 안건으로 빠르게 진행.

## 안건: [03] [CMS-2P] 선택된 상태의 LED 표시 수정

- 「임시회의-20250403」 [06] 전면 LED 안건(👉 Asis 유지)의 설계 재수정. 「결함회의-20250430」에서 ChainView에서 [CMS-2P] 선택 후 다른 모듈을 Set 버튼으로 선택 시, 새 모듈의 LCD Backlight Timeout 설정이 유지되지 않고 [CMS-2P]로 재선택되는 현상 발생.
- 원인: [CMS-2P]가 "선택된 상태"를 항상 Green 점등으로 표시하기 위해 3초마다 모듈로 선택 상태를 Write함 → 다른 모듈의 Set 버튼 신호를 무시하고 점등 신호 전송.
- Tobe: "선택된 상태" LED 동작을 메인의 "LCD Backlight Timeout [sec]" 설정값 기준으로 동작(예: 20초 설정 시 20초간 점등 후 종료). Green 점등(설정 시간 경과 후 종료)으로 변경, 나머지 LED 동작(통신상태 Orange·ID 설정오류 Red·이벤트 Red 점멸·Bootloader Orange 점멸)은 Asis 유지.

**👉 Tobe 안으로 결정** — 기존 Green 계속 점등 방식에서 LCD Backlight 설정 시간 경과 후 소등되는 방식으로 변경 완료.

## 안건: [05] 「분기 > SETUP > MEASUREMENT > Load Wiring」 "Load Direction" 설정에 따른 모듈별 누설전류 위상 변경 여부

> [이미지 image 10.png] [참고/메일 조국춘 2025-05-12] "[A2550] Load Direction(부하 방향) 설정의 영향 범위" — 1. CMZ/CMMZ(ZCT 내장): Load Direction 영향 범위는 전압 위상 기준 상전류 위상 180도 변경·ZCT 전류 위상 180도 변경. 2. CM 모듈: 상전류 위상 180도 변경·외부 ZCT 전류 위상 변경 없음(별도 독립 설치이므로).

- Asis: CMZ·CMMZ 변경되지 않음 / CMS+ZM 변경 / CM+외부ZCT 변경되지 않음.
- Tobe: CMZ·CMMZ 변경 / CMS+ZM 변경 / CM+외부ZCT 변경되지 않음.

**👉 Tobe 안으로 결정** — CMM·CMZ·CMMZ·CMSZ 템플릿 변경, 외부 ZCT만 Load Direction 설정에 따른 위상 변경 없음.
