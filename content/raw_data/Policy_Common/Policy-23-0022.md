# Policy-23-0022 [진행중] · Timeout [min] 팝업에 "Infinite" 체크버튼

## 결정 사항

- 「SETUP > CONTROL > Test Mode > Timeout [min]」 숫자패드팝업에 **"Infinite" 체크버튼 추가**(설정값 의미 명확화, Policy-23-0028 참고). 숫자패드 형태는 Policy-23-0024 따름.

> [이미지 Untitled 217.png] [Tobe] Timeout [min] 팝업. Range 1–1,440 + **Infinite 체크버튼**(해제 상태), 입력필드 60.

> [이미지 Untitled 218.png] [Tobe] SETUP > CONTROL > Test Mode 설정조회화면. Mode Off, **Timeout [min] = "Infinite"** 표시(체크 적용 시).

**「입력필드」 상황별 표시** (Entered=Enter 후 Apply 전 / Applied=Apply 후)
- 범위 외 값으로 적용된 상태에서 범위 내 값으로 변경 시: 의미상 가까운 최소/최대값 표시

> [이미지 Untitled 219.png] [예시] Infinite 체크→해제 시 입력필드에 **1,440**(최대값) 표시.

- 범위 내 값으로 적용된 상태에서 범위 외 값으로 변경 시: 실제값 표시, 다시 범위 내로 변경 시 임시보관 값 표시

> [이미지 Untitled 220.png] [예시] Infinite 해제→체크 시 입력필드 **0**(비활성, 회색 숫자패드).

> [이미지 Untitled 221.png] [예시] 다시 Infinite 체크→해제 시 **임시보관 값 10** 표시.

- 팝업 열릴 때 현재값 임시보관(Cancel 시 임시보관 값 복원, Enter 시 수정값 표시).

## 결정 배경

- 숫자패드만으로는 "0"의 의미(무한)를 정확히 표시하지 못함.

> [이미지 Untitled 222.png] [참고/통신맵] Test mode timeout(6002, UINT16, RW). **"범위 0, timeout 없음 / 범위 1–1440 / Default 60(min)"** — 0이 Infinite 의미.

- 참고: 숫자패드만 있는 다른 팝업은 범위 밖 입력 시 최소/최대값으로 자동변환(통신 입력도 동일).
