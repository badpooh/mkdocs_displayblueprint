Policy-23-0024 [진행중] · 숫자패드 수정팝업 범위기호·형태 통일

결정 사항

범위 표시 기호 통일
- 하이픈: "–"(엔 대쉬, en dash). 대외용 문서(장치화면/매뉴얼/홈페이지)는 "–" 기본, 대내용 문서(노션/스마트시트/설계문서)는 "~" 허용.

숫자패드 형태 통일 (버튼 표시 순서·Backspace 모양이 기존과 다름)
- Type 1 기본 / Type 2 선택적(숫자패드 상단에 En/Dis 선택항목 있을 때 등) / Type 3은 Password 전용

  • 동작: 음수는 "–" 먼저 입력. "." 버튼은 Range가 소수점 포함 시 Enable(입력필드에 "." 있으면 Disable), "–" 버튼은 Range가 음수 포함 시 Enable(입력필드 비어있지 않으면 Disable). 입력필드가 빈칸/"-"/"."/"−."만이면 「Enter」=「Cancel」 동작(2024/06/05 추가).
  • (2024/08) 디폴트 Disable 명확화, ".1"→"0.1"·"1."→"1.0" 자동 변환 허용 등 보완.

결정 배경

  • "~"는 전기분야에서 교류를 의미하고 UL에서 범위표시에 "~" 사용 금지 요청 → 문서는 "–". 숫자패드는 A7300 회의 결정을 A2500 회의에서 보완("±"→"–").
