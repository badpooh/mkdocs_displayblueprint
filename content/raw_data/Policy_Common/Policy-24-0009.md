# Policy-24-0009 · 데이터 수에 따른 콤보박스(페이징) 표시

## 결정 사항

- **데이터 0개**: 좌/우 화살표 Disabled, 중간영역 "None ∨" Disabled
- **데이터 1개**(1페이지 분량 미만, Asis 동일): 화살표 Enabled(Tap 시 변화 없음), 중간영역 "1/1 ∨" Enabled(팝업에 1번 버튼만)
- **페이지 수 고정**(예: RELAY>SETUP>PROTECTION>OCR, I/O>DI): 페이지 2개 이상일 때만 표시
- **페이지 수 가변**(예: RELAY>EVENT>EVENT LOG, A2550TEMP 분기 EVENT): 0개 이상이면 표시(0개면 Disabled)

> [이미지 Untitled 313.png] [예시/A2550TEMP] 분기 SETUP > EVENT > Temperature(ID10, TEMP+SENSOR 6). 우측 상단 **TEMP 콤보박스(페이징)**, Trigger/Threshold 40℃/Time Delay/DO Notification.

> [이미지 Untitled 314.png] [예시] 동일 화면 콤보박스 팝업 열림 — TEMP/Sensor.1~6 선택 버튼.

## 결정 배경

- EVENT 데이터 수에 따른 페이징 버튼 설계가 불명확. Asis-A7300은 데이터 0개여도 "0/0 ∨" Enabled(팝업 내 버튼 없음)로 표시.

> [이미지 Untitled 315.png] [참고/메일서비스] Nate·Bizbox — 데이터 0개여도 "1" 페이징 표시 + 이전/다음 화살표 Disabled.

> [이미지 Untitled 316.png] [참고/메일서비스] 페이징 형태(≪ < 1 > ≫). Naver/Kakao/Gmail은 0개 시 페이징 숨김.
