# Policy-25-0004 · ACL 기능 및 Telnet 차단

## 결정 사항

**ACL(Access Control List) 추가**
- 허용 리스트(White List) **10개 관리(ACL 1~10)** — 당초 5개(2025/03/20)였으나 LG U+ 요청(2025/10/27)으로 확장
- 통신매뉴얼: 공개 / 장치 화면: Enable/Disable만 제공(「SETUP > NETWORK > Advanced」, 0:Disable 디폴트 / 1:Enable) / AccuraSM: 상세 설정(활성화, IP Address, **Wildcard Mask**) 제공 — 일반 사용자모드 표시, 변경은 관리자·개발자 모드

> [이미지 image 28.png] [참고/A2500 Modbus] ACL Setup 레지스터. ACL setup access(3741), ACL(3742, 0:비활성 디폴트), ACL 1(3743), ACL 1 IP address(3744, 디폴트 10.10.10.0), **ACL 1 Wildcard mask(3746, 범위 0~32, 디폴트 24=0.0.0.255)**, ACL 2(3749)~.

**Telnet Enable/Disable 추가·디폴트 수정**
- 통신매뉴얼: 비공개 / 장치 화면: 미지원 / AccuraSM: 개발자모드에서만 제공 (0:Disable 디폴트 / 1:Enable)

- 이더넷 포트 있는 모든 장치에 적용. 단 **A7300은 기존 유지**(ACL 없음 — 고객 요청 시 재논의, Telnet은 SSH 사용이라 비대상).

## 결정 배경

- LG U+ 현장 보안 이슈로 ACL 요청. A3500에 기적용된 Telnet 차단 기능의 디폴트가 Enable이라 Disable로 수정 필요.

> [이미지 image 29.png] [참고] 김민수 이메일(2025-03-24). 임베디드SW파트 협의: **Subnet Mask 방식 → Wildcard Mask 방식으로 수정**, Release(양산)·Renewal 모두 ACL 추가. ACL 규격(IETF RFC 4949/3198)과 access-list 예시.

> [이미지 image 30.png] [참고] 김민수 이메일(2025-03-20). ACL에서는 Wildcard Mask가 일반적(서브넷마스크의 반전 — 255.255.255.0 ↔ 0.0.0.255) — ChatGPT/Perplexity 답변 비교 첨부.

> [이미지 image 31.png] [참고] "ACL에서 서브넷마스크 사용 예시?" 검색 — ACL은 Wildcard Mask 사용(Cisco access-list 100 permit ip 192.168.1.0 0.0.0.255 any).

> [이미지 image 32.png] [참고] 김민제 이메일(2025-10-27) "A3500F/E, A2500M ACL 기능 확장". LG U+ 요청으로 5→10개 확장, **ACL 운용 상태 확인 번지 추가**(A2500M: 0번 맵 2900번지, A3500F/W: 0번 맵 12900번지), AccuraSM 반영 필요.
