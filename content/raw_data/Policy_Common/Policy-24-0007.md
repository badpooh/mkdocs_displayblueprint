# Policy-24-0007 · 「EVENT」 Refresh/Read All/콤보박스(페이징) 버튼 표시·동작

> Policy-23-0032의 정책 추가.

## 결정 사항 — 표시 조건

| 버튼 | Shown | Enabled | Disabled | Hidden |
| --- | --- | --- | --- | --- |
| Refresh | 행 선택 해제 시 | Type 필터 리스트에 New 있을 때 | New 없을 때 | 행 선택 시 |
| Read All | 행 선택 해제 시 | 전체 필터(Type/Index/Date) 리스트에 Unread 있을 때 | Unread 없을 때 | 행 선택 시 |
| 콤보박스(페이징) | 행 선택 해제 시 | 전체 필터 리스트에 행 있을 때 | 행 없을 때 | 행 선택 시 |

## 결정 사항 — Tap 시 동작

- **Refresh**: ①데이터 새로고침 ②Index·Date 필터 From/To 체크 해제(변경값·정렬은 유지) ③1페이지 이동 ④세로 스크롤 맨 위
- **Read All**: 필터링된 행만 Read 처리
- **콤보박스 중간 영역**: 팝업 표시/숨김
- **콤보박스 좌/우 화살표**: 페이지 이전/다음(순환 구조 — 1페이지에서 좌 화살표 시 끝 페이지로)

## 결정 배경

- 기존 정책(Policy-23-0032)에서 Date/Index 필터링 상황의 버튼 표시·동작 설계가 불명확
- A2500/A3700N 이벤트는 Type 필터(부메뉴)+Read All만 존재. A7300은 Type(부메뉴)·Index·Date 필터(헤더, 정렬 포함). 공통: 메뉴 이동 시 데이터 새로고침.
- 「Read All」은 A7300 설계서 Rev1.00·정기회의-20230511에서, 「Refresh」·「콤보박스(페이징)」은 A7300 설계서 Rev1.01에서 추가됨(콤보박스는 이벤트 1000개 페이징).
