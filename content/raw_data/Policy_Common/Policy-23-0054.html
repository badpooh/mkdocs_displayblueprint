Policy-23-0054 · 「SETUP」 Command 타입 항목 표시 정책

결정 사항

  • Command 타입을 2가지로 구분 ("Command 1"은 기존 Command와 동일, "Command 2" 추가하며 기존을 "Command 1"로 정정)
구분 Command 1 Command 2
조회 상태 항목값 공백 공백
항목 Tap 시 확인팝업 열림 변경팝업 열림
조회 시 팝업 디폴트 선택 없음(Confirm/Cancel 모두 Highlight 없음) 없음(리스트 Highlight 없고 Enter Disabled, 선택 시 Highlight·Enter Enabled)
Entered 상태 항목값 공백 선택된 항목 표시
Entered 시 팝업 디폴트 없음 이전 선택 항목 Highlight, Enter Enabled
Applied 상태 조회 상태와 동일 조회 상태와 동일
  • Command 1 예: 「A2500 > SETUP > DEVICE > Module ID」, 「CONTROL > Data Reset」·「Demand Sync」
  • Command 2 예: 「A2500 > SETUP > DEVICE > MCCB Status > All MCCBs Status1 (DI.1)」, 「A2550IO-DO > SETUP > I/O > DO > Control」(2024/02/19 추가)
  • 공통: 항목값 Bold, 변경(Entered) 시 바탕색 Highlight, 항목 Tap 가능=항목명 검정/흰색·불가=진회색

색상 정의

상태 항목명 항목값 바탕색
Normal 검정 검정 연회색 #DEDEDE
Disabled 진회색 #B1B1B1 진회색 연회색
ReadOnly 진회색 검정 연회색

Disabled & 변경 진회색 진회색 연청색 #909AA7
ReadOnly & 변경 진회색 흰색 연청색
Command 1 & 변경 흰색 - 진청색
Command 2 & 변경 흰색 흰색 진청색

결정 배경

  • Command 타입 표시를 2가지로 분리해 동작 정의 필요. "Command" 타입 = 일회성 명령용, Applied 이후 장치 상태와 무관(상태 반영 없음).
