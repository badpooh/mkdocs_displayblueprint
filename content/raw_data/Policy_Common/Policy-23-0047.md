# Policy-23-0047 · EVENT > MEASUREMENT 부메뉴(Etc./All) 정책

## 결정 사항

- 「EVENT > Dashboard」에서 주메뉴 이벤트 총수량 = 부메뉴들 이벤트 총합계여야 함.
- **「All」 부메뉴**: 모든 모듈에 표시(기존 유지). 이벤트 분류가 아니라 모든 이벤트를 시간순 등으로 보는 편의 목적 — Dashboard에 분류로 표시되지 않음.
- **「Etc.」 부메뉴**: 모든 모듈에 표시. 단 메인/분기에 따라 포함 이벤트가 다름:
  - [메인] 기본 제공 항목(Offset 미지정: 전압·전류·전력·누설전류) 외의 Offset 지정 이벤트 + 분류 미제공 기타 이벤트(예: TEMP 이벤트)
  - [분기] 기본 제공 항목(전압·전류·전력·누설전류·온도·습도) 외의 Offset 지정 이벤트 + **Legacy 이벤트**
    - 단 메인에서 Legacy 이벤트는 Etc.로 이동하지 않고 기존 분류 유지(예: CMZ→CM 변경 후에도 Leakage 이벤트는 「메인 > EVENT > MEASUREMENT > Leakage」에 표시)
- 부메뉴가 「All」·「Etc.」만 있으면 두 화면의 이벤트 리스트는 동일.
- Dashboard의 분모(총저장가능수량 "100"/"1000")는 폰트 작게 표시.
- 예(A2500) 모듈별 부메뉴: [A2500M] 현재 유지 / [CMZ·CMS+ZM·CM+외부ZCT·CMMZ] All·Current·Power·Leakage·Etc. / [CM·CMS단독·CMM] All·Current·Power·Etc. / [TEMP] TSEN만: All·Temperature·Etc., THSEN 포함: All·Temperature·Humidity·Etc. / [GW·GWM·IO] All·Etc.

## 결정 배경

- 용어 "Legacy 이벤트": 현재 연결상태와 다른 이전 연결상태에서 발생한 이벤트(모듈 ID 유지·Type만 변경, 또는 부속모듈 연결 해제).
- 예) ID 2 장치를 [TEMP]→[CMZ]→[CM]으로 Type만 변경한 경우:

> [이미지 Untitled 351.png] [참고/분기] EVENT > MEASUREMENT > All — Legacy 이벤트(ID 2의 Temp Over index 2111, Leakage Under index 2120)가 **All에만** 표시됨.

> [이미지 Untitled 352.png] [참고/메인] EVENT > MEASUREMENT > **Etc.** — ID 2의 Temp 이벤트(index 2111 "TSEN1 Start Value 27.2°C" 등)가 Etc.에 표시됨.

> [이미지 0.png] [참고/메인] EVENT > MEASUREMENT > **Leakage** — ID 2의 Leakage Under 이벤트(index 2120 "Due to module disconnected" 등)가 기존 분류(Leakage)를 유지함.

- 예) [CMS+ZM]에서 [ZM] 연결 해제: 분기에서 부메뉴 「Leakage」는 숨김 처리되나 Leakage 이벤트는 「All」에만 표시됨.

> [이미지 2.png] [참고/분기 ID01 CMS-3P4W-60A] EVENT > MEASUREMENT — 부메뉴 All/Current/Power만 표시(Leakage 숨김), All에 Leakage Over 이벤트(index 2092 등) 표시.

- 배경: 「분기 > EVENT > Dashboard > MEASUREMENT」 수량과 하위 분류 합계가 다른 경우 발생 — Legacy 이벤트·「사용자정의 > Etc.」 이벤트가 「All」에만 포함되는데 All은 Dashboard에 표시되지 않기 때문 → Etc. 부메뉴로 해결.
