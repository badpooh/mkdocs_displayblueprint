# Policy-23-0046 · 장치 버전 표시 통일

## 결정 사항

- **"Version"/"Revision" 구분**: "Hardware Revision"에만 Revision 사용(PCB 보드 Pin 정보를 읽어 자동 구분되는 경우만). 그 외 모두 "Version".
- **"Firmware"/"Software" 구분**: Firmware = 장치 프로그램(Application·Kernel·Bootloader를 묶는 상위 용어 — "Firmware Version"은 이 셋으로 구성). Software = 컴퓨터 프로그램(AccuraSM·AccuraDM 등).
- 통일된 화면 구성: 공통 항목 선배치(Serial Number, Hardware Revision), 장치별 특이항목 후배치. **PCB Version 화면 삭제**(출고 테스트 문서에 기록되나 장치 입력 절차 미명시, A3700N 모듈은 Map에 항목 없음). **MAC Address 삭제**(SETUP에서 ReadOnly 표시 중). **버전은 하나로 통합 표시**(한 CPU에 여러 App이 있어도).

> [이미지 Untitled 223.png] [Tobe/수정안 xlsx] 제품별 시스템정보 표시안 — A2500M/A2500D/A2550/A3700/A3700모듈/A7300_METER/RELAY 공통: Serial Number, Hardware Revision(22.01), **CPU Firmware Version "A1.5.004, K1.0.005, B1.0.008"**, **MCU Firmware Version "A1.5.004, B1.2.001"** 통합 표기.

- MCU Firmware Version 버전체계: **메인장치 5자리**(예: A1.5.004 — A2500M/D·A3700N·A7300), **분기장치(서브모듈) 기존 3자리 유지**(예: A180, A2550의 5자리 전환은 추후 검토).
- Hardware Revision 버전체계: **메인장치 "00.00" 형태**(예: 22.01), **분기장치 정수 유지**(예: 2). 분기의 MCU Hardware Revision은 표시하지 않음(필요 시 AccuraSM에서).
