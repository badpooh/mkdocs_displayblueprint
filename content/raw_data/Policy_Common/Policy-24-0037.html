Policy-24-0037 · SW 버전 부여 정책 (대표버전)

결정 사항

  • 제품별 대표버전 사용: SW 개발계획·테스트는 대표버전 기반(컴포넌트 버전 전부 표시 곤란). 컴포넌트 버전이 오르면 대표버전도 같이 올림.
  • 대표버전 담당: 주로 Main 장치 Application(상위 CPU+MCU 구성이면 상위 CPU의 Main App, Main/UI 분리 시 상위 통신 담당 SW, MCU 단독 제품은 MCU App).
  • CPU 有 제품: CPU App 버전(예: A2500M의 MCU FW나 Bootloader만 변경돼도 CPU App 버전 올림). CPU 無 제품: MCU FW 버전(예: A2550 Bootloader만 변경돼도 MCU FW 버전 올림).
  • 항목명: CPU 대표버전 제품 "Application Version", MCU 대표버전 제품 "Firmware Version".
  • 버전 형태: [Major / Minor / Patch / Develop / Internal Patch / Build 횟수]
  • Major(대규모·호환성 변경) / Minor(기능 추가·소규모) / Patch(버그 수정).
  • Develop: 미릴리즈 테스트버전 번호 — 내부 한정·외부유출 금지(commit 안 함, 중복 가능). 현장 긴급 대응으로 외부 전달 시엔 develop까지 commit.
  • Internal Patch: 비공개 소규모 개선 — Major/Minor/Patch 유지한 채 수정, 최종 고객 비공개, SM 개발자모드에서 확인, Updater 인식 가능.
  • Build 횟수: 테스트 이관 시도마다 부여 — "최종버전(build 포함) 부여 후 테스트 이관 → 통과 → 생산이관"으로 절차 개선(리빌드 오류 예방). 최초 0, 재시도마다 +1, 코드 수동 변경·commit.
  • 공개 여부: Major.Minor.Patch 화면·SM 공개 / Develop 0이면 비공개 / Internal Patch 화면 비공개·SM 개발자모드 공개 / Build는 버전과 분리된 보조정보.
  • 주의사항: 개발팀을 벗어나는 버전은 반드시 버전관리 툴에 commit. 테스트팀 통과 후 연구부를 벗어난 버전 번호는 재활용 불가(생산이관·현장 적용 포함). 동일 버전 유지 필요 시 Internal Patch 증가.
  • 부여 방법: 릴리즈 시 변경된 요소별 개별 버전 부여, 대표버전 SW는 자신의 변경이 없어도 신규 버전 부여. 화면 표시는 대표버전, 각 SW 요소는 개별 버전 유지.
  • 버전 데이터: 상위 CPU 대표버전 SW는 6개 요소 각 16bit. MCU는 Major/Minor/Patch 합산 16bit(0~65535) — 해석: Major 1자리/Minor·Patch 각 2자리(10103 → 1.01.03 → 표기 1.1.3). 기존 1.0.103 해석은 Patch 영역 과다 문제. 기존 1.1.000 장치는 1.10.00으로 전환(버그패치 1.10.1, 기능개선 1.11.0). Develop/Internal Patch/Build는 각 1byte 추가 공개.
  • AccuraSM 표시: Information 화면(현행 유지). Normal/Admin Mode는 Major.Minor.Patch.Develop. Developer Mode에서 Internal Patch("p")·Build("b") 추가 표시.
  • MCU 대표버전(MCU Only) 제품은 Hex파일 0x0210 위치에 [Develop, Internal Patch, Build] 기록:
  • MCU 버전 형태를 1.xx.xx로 변경.
  • 버전 불일치 진단·알림 기능 추가: 부팅 시 1회, 장치 내 단위 프로세서(CPU Kernel/Bootloader, MCU(M)/(R)/(IO) FW/Bootloader) 버전이 Package 버전 정보와 불일치하면:
  • 이벤트 알림(&Log)은 하지 않음.

결정 배경

  • 개발→테스트→생산이관 과정의 오류 절감 필요. 현재 문제: 한 제품에 SW 요소가 다수(A2500M 6개, A2500M/D 합산 9개)로 관리 복잡, 현장 버전 확인 시 모든 요소 확인 필요. 제품별 버전 표시 공간이 달라 테스트 요청 버전도 제각각.
  • 장치별 SW 버전 표시 현황:
  • 통신맵 버전 영역 현황:
  • A3700N은 현장 업데이트 제한으로 비공개 Patch 버전 사용 중:
  • 피앤티팀 테스트 완료 FW를 버전 변경·리빌드 후 배포하는 문제. 예: A2300N 테스트 2.0.14→배포 1.0.15, A2500 1.10.1.999→1.11.0, A2550CM 9.6.0→1.6.1, A3700N 1.7.0p3→1.7.0p4.
