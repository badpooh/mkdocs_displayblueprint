# Policy-24-0034 · I/O 이벤트 로그 채널별 기록

## 결정 사항

- I/O 상태 변경 이벤트를 **채널별로 기록·표시**. 여러 채널이 동일 시점에 변경되면 "Time" 값을 동일하게 기록.
- 예시-A7300 이벤트 화면:

> [이미지 image 4.png] [Tobe/A7300 RELAY] EVENT LOG > All — 채널별 기록: `<CB.Off> Open`, `<DO.4> Closed`, `<CB.Off> Closed`, Trip Command(Source: Protection function), OCR1(Def.) Operate: Phase A, `<DI.1> Closed`.

> [이미지 image 5.png] [Tobe/A7300] EVENT LOG > I/O(CB) — `<CB.ON>`/`<CB.Off>` 채널별 Open/Closed 기록.

> [이미지 image 6.png] [Tobe/A7300] EVENT LOG > I/O(DI·DO) — `<DO.1> Open`, `<DO.4> Closed`, `<DI.1> Closed` 등 채널별 기록.

> [이미지 image 7.png] [Tobe/A7300] EVENT LOG > Control — "DO.1 Off Command / Source: I/O Test" 형태(CB·Trip Command와 유사 형태로 수정).

- 분류별 상세내역(설계 표):
  - All: CB/DO/Control/Protection 등 — `<CB.Off> Open`, `<DO.4> Closed`, Trip Command(Source: Protection Function), OCR1(Def.) Operate: Phase B.
  - I/O: `<CB.Off>`, `<DO.1>`, `<DI.1>`, `<HW Fault>` 채널별 Open/Closed. **A7300IO는 Type에 "IO-" Prefix + 채널 Type**(IO-DI, IO-DO, AI, AO, RTD), Description은 `[ID 1] (IO-DIO) <DI.1> Open` 형식.
  - Control: `<DO.1> On/Off/Normal Command, Source: I/O Test` — Normal Command는 Test(On·Off / Open·Closed) 상태에서 Normal로 변경 시에만 기록. `<DI.1>`, `<HW Fault>` 동일.

## 결정 배경

- A7300 이벤트 내역의 I/O 상태 표시 방식 개선 의견(2024/11/05, 안창모): A3700과 동일하게 채널별 기록 필요.

> [이미지 clipboardImage_24_1105_161628_002.jpeg] [참고] 안창모 카톡(2024-11-05) — A3700/A7300 이벤트 화면 비교 캡처와 함께 "A3700은 채널별로 기록되도록 되어있습니다. 정책으로 반영해야될 것 같습니다."

> [이미지 clipboardImage_24_1105_162613_291.jpeg] [참고] 첨부 비교 이미지 — A3700(장치 사진): `DIO [ID 1] [CH 1] Open/Closed` 채널별 기록 vs A7300(Fig 5.76): DO 이벤트가 `On:[00000000] Off:[00000001] Status:[01000000]` **Bit 형태**.

- 장치별 표시 방식 상이:

> [이미지 A3700N_20241106_095115.png] [Asis/A3700N] EVENT > MEASUREMENT > Module — `DI [ID 4] [CH 1] Open/Closed` **채널별 표시**.

> [이미지 image 8.png] [Asis/A7300] EVENT LOG > All — CB/DO 이벤트가 `On:[00] Off:[01] Command Status:[00]` **Bit 형태 표시**.

> [이미지 image 9.png] [Asis/A7300] EVENT LOG > I/O(CB) — Bit 형태(On/Off/Command Status).

> [이미지 image 10.png] [Asis/A7300] EVENT LOG > I/O(DO·DI) — Bit 형태(DI는 Status 비트).

> [이미지 image 11.png] [Asis/A7300] EVENT LOG > Control — "DO Test Command Mode:[000 0001] Status:[000 0001]" Bit 형태.
