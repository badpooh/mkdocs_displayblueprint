# Policy-24-0023 · 대량 이벤트 "Mass" 표시

## 결정 사항

- 대량 이벤트 발생 상황을 「이벤트 알림창」과 「EVENT > Dashboard」에 표시 추가.

> [이미지 image 121.png] [Tobe/A7300] 이벤트 알림창에 **빨강 바탕 "Mass"** 표시 + R 003 — RELAY 영역 대량 이벤트(HOME PROTECTION 단선도 화면).

> [이미지 image 122.png] [Tobe/A7300] M 002 + **Mass** — METER 영역 대량 이벤트.

> [이미지 image 123.png] [Tobe/A7300 METER] EVENT > Dashboard. POWER QUALITY에 **Mass 아이콘 + 298**, Dip 336(298 New) — 기존 New Event Count 표현 유지.

> [이미지 image 124.png] [Tobe/A7300 RELAY] EVENT Dashboard. EVENT LOG에 **Mass + 298**, Protection 336.

## 결정 배경

- A7300 이벤트 저장: METER POWER QUALITY·RELAY EVENT LOG는 휘발메모리 기록 후 DB 저장(대량 이벤트 가능성 높음), FAULT RECORD·SYSTEM은 DB 직접 저장.
- 대량 이벤트 발생 시 DB 저장을 잠시 멈추고 휘발메모리에만 기록. 판단: 이벤트 구간 Count ≥ 3 (RELAY 초당 80개·METER 초당 25개 초과 시 +1, 이하 시 -1; 발생 후엔 초과 카운팅 안 함, Count 0이면 해제). 해제 시 1000개를 10개씩 Delay 두고 DB 재저장(약 1분).
- 대량 발생 중엔 DB에 로그가 없어 화면에 NewEvent 미표시 → 사용자가 정상 동작으로 오인. 이를 표시할 필요.
