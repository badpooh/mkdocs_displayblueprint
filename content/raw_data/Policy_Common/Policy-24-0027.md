# Policy-24-0027 · 문자열 항목 최대표시문자수 통일

## 결정 사항

- 설정값이 문자열("Char")이고 가로 길이가 제한된 항목의 **최대표시문자수를 "20자"로 장치간 통일**. 대상: 「SETUP > SYSTEM > Description > Device Name」, 「User-Defined Info」, 「Location」, 「Load Name」.
- 초과 시 앞에서부터 **"20자+…"** 표시(총 23자). 예: Device Name 최대설정 30자에 21자 이상 입력 시 설정조회화면에 20자+… 표시.
- 가변폭폰트(Roboto) → **고정폭폰트(DejaVu Sans Mono)**로 수정. 단 「HOME > SYSTEM > Description」은 가로 길이 제한이 없어 기존 Roboto 유지:

> [이미지 A2500_20240819_163243.png] [참고/A2500] HOME > SYSTEM > Description — Device Name "WWW…"(30자)·User-Defined Info(30자)가 줄임 없이 전체 표시(가로 제한 없음, Roboto 유지).

- 장치별 화면 수정 예시:

> [이미지 Untitled 391.png] [Tobe/A2500 메인] SETUP > SYSTEM > Description — Device Name·User-Defined Info가 고정폭 폰트로 **20자+"…"** 표시(Location, Installation Year/Month/Day 포함).

> [이미지 Untitled 392.png] [Tobe/A2500 분기 CMZ-3P-60A] SETUP > SYSTEM > Description — Load Name·User-Defined Info 20자+… 표시.

> [이미지 image 116.png] [Tobe/A7300 METER] SETUP > SYSTEM > Description — Device Name·User-Defined Info 20자+…(고정폭, 초록 테마). A7300 화면은 임시회의-20250123 내용 참고.

> [이미지 image 117.png] [Tobe/A3700N] SETUP > SYSTEM > Description — 동일하게 20자+… 표시.

## 결정 배경

- 「A2500-UI기획 > 정기회의-20230202」 "글자수 표시 통일 및 폰트 문제": SETUP 화면 기준 통일, 13자 초과 시 10자+…(총 13자) 안과 고정폭 폰트 후보(D2Coding/Consolas/DejaVu) 검토 → 기존 폰트 사용·최대 글자수 통일로 정리.
- 「A2500-UI기획 > 정기회의-20230216」: **고정폭 폰트(DejaVu Sans Mono) 사용 결정** — 가로 길이 제한된 곳(Dashboard Location, StatusView Feeder 정보 등 최대 12자)에만 제한적 사용, Description은 기본 폰트.
- 문자열 항목의 최대표시문자수가 장치별로 상이하고, 가변폭폰트라 가로폭 넓은 문자(W 등) 입력 시 값이 잘리거나 다른 영역을 침범:

> [이미지 image 118.png] [Asis/A2500 메인] Description — Device Name "W…"가 표시영역 초과로 **앞부분이 잘려 표시**(빨간 박스), User-Defined Info도 앞이 잘림.

> [이미지 image 119.png] [Asis/A2500 분기 CMZ-3P-100A] Description — Load Name·User-Defined Info 동일하게 앞부분 잘림(빨간 박스).

> [이미지 image 120.png] [Asis/A7300 METER] Description — 20자 초과 시 20자+… 표시이나 가변폭 폰트로 Device Name이 좌측 부메뉴 영역까지 침범(빨간 박스).

> [이미지 A3700N_20240805_145250.png] [Asis/A3700N] Description — **15자+…(총 18자)** 표시로 A7300(20자)과 상이.
