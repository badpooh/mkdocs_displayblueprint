# Policy-23-0021 · Event LED Hold Time 팝업 수정

## 결정 사항

- 「SETUP > SYSTEM > LED > Event LED Hold Time [sec]」 팝업: 숫자패드 상단에 **탭버튼(Continuous/Self-resetting/Definite Time) 추가** — 설정값(0/60000)의 의미를 정확히 전달. 탭버튼 항목명은 「이벤트 DO」와 동일(Event LED는 이벤트 DO와 유사 동작, Policy-23-0028/0029 참고). 숫자패드 형태는 Policy-23-0024 참고.

> [이미지 Untitled 376.png] [Tobe/A2500] Event LED Hold Time [sec] 팝업 — 상단 탭 Continuous/Self-resetting/**Definite Time**(선택), Range 1–59,999, 입력필드 59,999.

> [이미지 Untitled 377.png] [Tobe/A2500] SETUP > SYSTEM > LED 설정조회 — Ethernet/Module/Event LED Period·On time + **Event LED Hold Time = "Continuous"**(숫자 대신 의미 표시).

- 「입력필드」 상황별 표시(실제 범위: 0=Self-resetting, 1~59999=Definite Time, 60000=Continuous, 디폴트 60000):
  - 팝업 열릴 때 현재값 임시보관(Cancel 시 복원, Enter 시 수정값 표시). "Entered"(Enter 후 Apply 전)/"Applied"(Apply 후).
  - 범위 외 값 적용 상태에서 범위 내로 변경: 의미상 가까운 값 표시 — Continuous→Definite Time이면 59,999, Self-resetting→Definite Time이면 1.

> [이미지 Untitled 378.png] [Tobe] Definite Time 선택 시 — 입력필드 59,999(Continuous에서 전환된 경우 최대값 표시).

  - 범위 내 값(예: 60) 적용 상태에서 범위 외로 변경: 실제값 표시, 다시 범위 내로 변경 시 임시보관 값 표시.

> [이미지 Untitled 379.png] [Tobe] **Continuous 탭 선택** — 입력필드 60,000(회색), 숫자패드 Disabled.

> [이미지 Untitled 380.png] [Tobe] **Self-resetting 탭 선택** — 입력필드 0(회색), 숫자패드 Disabled.

> [이미지 Untitled 381.png] [Tobe] 다시 Definite Time 선택 — 임시보관 값 **60** 복원 표시.

- 매뉴얼 기준으로 구현 수정 → 이벤트 DO와 이벤트 LED 동작을 유사하게:

> [이미지 Untitled 382.png] [기준/A2500 통신매뉴얼] 3928 Event LED hold time — 0: 이벤트 종료 시 LED 꺼짐 / 1–59,999: 설정 시간 유지 후 꺼짐 / 60,000: 사용자 해제 시까지 무제한 유지.

## 결정 배경

- A7300 회의(정기회의-20220419/0421)에서 논의됐으나 A7300에는 Event LED가 없어 A2500·A3700 적용으로 정리.
- 숫자패드만으로는 "0"과 "60000"의 의미를 표시 못함:

> [이미지 Untitled 383.png] [Asis] 스마트시트 Modbus Map 3928 — 범위 0(이벤트 종료 시 LED 꺼짐)/1–59999/60000(=Infinite, 사용자 확인 시 꺼짐), Default 60000(infinite).

- 참고(DO Setup 매뉴얼 — 탭버튼 명칭의 근거):

> [이미지 Untitled 384.png] [참고/A2500 Modbus Map] 6607 Hold time(1~59,999sec), 6608 Hold mode(**0: Continuous, 1: Self-resetting, 2: Definite-time**).

> [이미지 Untitled 385.png] [참고/A2500 통신가이드] 6607 DO hold time — "범위 0: 이벤트 종료 즉시 자동해제…" ← **"0" 부분 삭제 & 장치화면 0 입력불가로 수정 요청**(2023/04/18 기술문서 Task 등록). 6608 DO hold mode 0/1/2.

> [이미지 Untitled 386.png] [참고/A3700N Modbus Map] 124 Hold time of DO Channel 1(Definite Time일 때, 1–59999), 125 Hold mode(0/1/2).

> [이미지 Untitled 387.png] [참고/A3700N 통신가이드] 124/125 동일 — Continuous(사용자 리셋까지)/Self-resetting(종료 시 자동)/Definite-time(설정 시간 후).

- 매뉴얼과 실제 장치 동작 불일치(A2500·A3700N 매뉴얼 내용은 동일):
  - A2500: "0" 설정 시 실제 구현은 **LED 동작 안 함**.

> [이미지 Untitled 388.png] [Asis/A2500 Modbus Map] 3928 — 매뉴얼상 0은 "이벤트 종료 시 LED 꺼짐"이나 실동작과 다름.

  - A3700N: "0" 설정 시 실제 구현은 **LED 1초 동작**.

> [이미지 Untitled 389.png] [Asis/A3700N Modbus Map] 3928 — Range 0–59999/60000(=Infinite), Default 60000(=Infinite).

> [이미지 Untitled 390.png] [Asis/A3700N 통신가이드] 3928 — 0: 이벤트 종료 시 꺼짐/1–59,999/60,000: 무제한 유지.

- 참고: 숫자패드만 있는 다른 팝업은 범위 밖 값 Enter 시 최소/최대값 자동변환(통신 입력 동일).
