# Policy-24-0032 · Ethernet DHCP 화면 표시 정책

## 결정 사항

- IP Address/Subnet Mask/Gateway 항목에 DHCP(Enable/Disable)에 따라 **Static/Dynamic 정보** 표시:
  - DHCP "0: Disable": Static 정보(Asis 동일).
  - DHCP "1: Enable": **Dynamic 정보 + 3항목 ReadOnly** 표시.

> [이미지 image 42.png] [참고/AI 답변] DHCP 서버 장애 시 PC 동작 — APIPA(169.254.0.1~169.254.255.254) 자동 할당.

> [이미지 image 43.png] [참고/AI 답변] APIPA 시 Subnet Mask는 항상 **255.255.0.0**, Gateway는 **0.0.0.0**(게이트웨이 미설정 의미).

> [이미지 image 44.png] [참고/AI 답변] 0.0.0.0의 의미 — "지정되지 않은 주소", DHCP 클라이언트가 IP 받기 전 임시 사용 등.

> [이미지 image 45.png] [Tobe/A3700N] DHCP Enable 정상 — IP 10.10.26.142/Subnet 255.255.224.0/Gateway 10.10.26.1 **Dynamic 값, ReadOnly(회색)** 표시.

  - DHCP가 정상 동작하지 않는 경우(예: 통신연결 문제): IP·Gateway "0.0.0.0", Subnet Mask "255.255.0.0" 표시.

> [이미지 image 46.png] [Tobe/A3700N] DHCP Enable + 서버 미응답 — IP/Gateway **0.0.0.0**, Subnet **255.255.0.0**, ReadOnly.

- **DHCP 변경 Entered 상태 표시**: Disable→Enable 시 "0.0.0.0"(&255.255.0.0)+ReadOnly+Highlight, Enable→Disable 시 Static 값+Normal+Highlight. **Highlight는 설정조회화면 대비 값이 변경된 항목에만** 적용:
  - 예1) Disable→Enable(Entered)→다시 Disable: 1단계에서 4항목 모두 Highlight, 2단계에서 원상복구(Highlight 없음).

> [이미지 image 47.png] [예1-0·2/A3700N] 설정조회 — DHCP Disable, IP 10.10.10.100/255.255.255.0/10.10.10.1 Normal.

> [이미지 image 48.png] [예1-1/A3700N] DHCP Enable(Entered) — DHCP·IP(0.0.0.0)·Subnet(255.255.0.0)·Gateway(0.0.0.0) **모두 Highlight+ReadOnly**, Apply/Cancel 표시.

  - 예2) Disable→Enable인데 기존 Subnet이 이미 255.255.0.0인 경우: **Subnet은 값이 같으므로 Highlight 없이 ReadOnly만**.

> [이미지 image 49.png] [예2-0/A3700N] 설정조회 — Subnet **255.255.0.0** 설정 상태.

> [이미지 image 50.png] [예2-1/A3700N] Enable(Entered) — IP·Gateway는 Highlight, **Subnet은 ReadOnly만**(값 동일).

  - 예3) Enable→Disable(Entered)→다시 Enable: Static 값으로 Highlight 후 원상복구.

> [이미지 image 51.png] [예3-1/A3700N] Disable(Entered) — IP 10.10.10.100/255.255.255.0/10.10.10.1 **Static 값+Highlight**(ReadOnly), Apply/Cancel.

  - 예4) Enable→Disable인데 Gateway Static 값이 Dynamic 값과 동일(10.10.10.1)한 경우: **Gateway는 Highlight 없음**.

> [이미지 image 52.png] [예4-0/A3700N] 설정조회 — DHCP Enable, Gateway 10.10.10.1(Dynamic).

> [이미지 image 53.png] [예4-1/A3700N] Disable(Entered) — DHCP·IP·Subnet Highlight, **Gateway는 값 동일로 Highlight 없음**.

- 💡 A2500은 현재(2024/10/24) DHCP 기능 없음 — 추후 현장 요청 시 추가.

## 결정 배경

> [이미지 image 54.png] [참고/A3700N Modbus Map] 3722 DHCP(0 Disable/1 Enable), **3723 IP address·3725 Subnet mask·3726 Gateway(노랑) = DHCP 서버에서 받은 Dynamic 값**(R 전용, 통신맵에만 존재).

- Asis: IP/Subnet/Gateway 항목이 DHCP 설정과 무관하게 **Static 정보만 표시**(Dynamic 정보는 화면 미표시):

> [이미지 image 55.png] [Asis/A3700N] DHCP **Disable** — Static 정보 표시(IP 10.10.10.100 등), DHCP 항목 빨간 박스.

> [이미지 image 56.png] [Asis/A3700N] DHCP **Enable** — 여전히 Static 정보 표시(실제 할당 IP와 무관) → 개선 필요.
