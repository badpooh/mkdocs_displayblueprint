# Policy-23-0028 · "Infinity" → "Continuous"/"Infinite"

## 결정 사항

- 사용처에 따라 용어 구분:
  - **Event LED는 "Continuous"** — DO와 동작이 유사하므로 같은 이름 사용("출력"에 의미).
  - **Timeout은 "Infinite"** — "시간"에 의미.

> [이미지 Untitled 37.png] [Tobe] Event LED Hold Time [sec] 팝업 — 상단 토글 **"Continuous"**/Self-resetting/Definite Time, Range 1–59,999.

> [이미지 Untitled 38.png] [Tobe] Timeout [min] 팝업 — Range 1–1,440 + **"Infinite" 체크박스**.

## 결정 배경

- "A7300-UI 기획"에서 명사형이 좋다는 의견으로 "Infinite"→"Infinity"로 수정했었음. 「A2550CMS/ZM 정기회의-20230406」에서 "Infinity"는 "무한대" 의미가 강해 재검토 요청 → 「정기회의-20230420」 결정.
- Asis: 대부분 장치는 "Infinite"(문서·화면), A7300 화면기획만 "Infinity", AccuraSM은 "No timeout". 사용처:

> [이미지 Untitled 39.png] [Asis] Demo Mode > Timeout [min] 팝업 — **"Infinity" 체크박스** 표기.

> [이미지 Untitled 40.png] [Asis] Event LED Hold Time [sec] 팝업 — Disable/Enable 토글 + **"Infinity" 체크박스** + Range 1–59,999.

> [이미지 Untitled 41.png] [Asis/A3700N] DO Type 팝업 — Type(Latch/Pulse) + **Hold Mode(Continuous/Self Resetting/Definite Time)**.

> [이미지 Untitled 42.png] [참고] DO Type 설정표 — Type 0:Latch/1:Pulse, Hold Mode 0:**Continuous**/1:Self Resetting/2:Definite Time(디폴트 Continuous).

- 용어 후보: Infinite(무한의), Infinity(무한, 명사형), Unlimited, Endless, Boundless, No timeout, Continuous(계속되는).
- "Infinite"가 영문 문서에 많이 사용됨(강경아):

> [이미지 Untitled 43.png] [참고] Cisco 설정 문서·계측기 매뉴얼 — accept-lifetime/send-lifetime의 "**infinite**" 키워드, Hold Time "0 = **infinite**" 사용 예.

> [이미지 Untitled 44.png] [참고] PCR 장비·보호계전기 매뉴얼 — "**infinite** hold", Setting table Range "0.5...2.5; **infinite**" 사용 예.

- Tobe 검토: Event LED를 DO와 같은 이름으로(Reset 시점만 조금 다름) — 1안 "출력" 의미 **"Continuous"**(채택), 2안 "시간" 의미 "Infinite Time":

> [이미지 Untitled 45.png] [참고/2안] Event LED Hold Time 팝업 — 토글 "**Infinite Time**"/Self-resetting/Definite Time (미채택안).

- 👉 Event LED: 1안 "Continuous"(DO·LED 용어 통일), Timeout: "Infinite"로 결정.
