# Policy-24-0036 · 통신 포트별 호환맵 설정 정책

## 결정 사항

- **모든 기존 연결에 변경사항 즉시 적용 (A3700N 방식)**

## 결정 배경

- 통신 포트별(Port 502/503, RS-485) 호환맵 설정 변경 적용 방식이 장치별로 상이:

| 장치 | 적용 방식 | 문제상황 |
| --- | --- | --- |
| A2500 | 변경한 연결만 즉시 적용, 기존 연결은 재연결 시 새 맵 | 사용자가 전체 적용으로 오인 → 혼란 |
| A3700N | 모든 기존 연결에 즉시 적용 | 테스트용 일시 변경 시 다른 연결도 끊길 수 있음 |

> [이미지 image 37.png] [참고] 문부기 이메일(2024-06-07) "A2500/A3700N 포트별 맵 적용 정책 결정 필요".

> [이미지 image 38.png] [참고] 첨부 비교 화면 — A2500/A3700N의 SETUP > NETWORK > Advanced 맵 설정 비교.

> [이미지 image 39.png] [참고] 현재 상황 정리. A2500=변경한 연결만 즉시(재연결 시 새 맵 — 예: A2300 맵이지만 A2500 맵 주소를 보게 되는 문제), A3700N=모든 연결 즉시(Display도 하나의 연결이라 재연결 필요 문제). 결정 필요 사항 1안=모든 연결 즉시 / 2안=변경 연결만.

> [이미지 image 40.png] [참고/A2500] SETUP > NETWORK > Advanced. **Port 502 Map(Accura 2500)·Port 503 Map(Accura 2300)·RS-485 Map(Accura 2500)**(빨강) — 기존 장치(A2300) 호환용 포트별 맵 설정.

> [이미지 image 41.png] [참고/A3700N] SETUP > NETWORK > Advanced. Port 502/503·RS-485 Map 모두 "Accura 3700 TET LCD"(빨강).
