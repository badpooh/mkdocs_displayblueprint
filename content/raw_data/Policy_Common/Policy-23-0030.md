# Policy-23-0030 · PF "No Load" 표시 정책

## 결정 사항

- 「SETUP」 항목명·값 용어 통일: **PF Sign**(0: Unsigned / 1: Signed 디폴트), **PF Value at No Load**(0: PF = 0 / 1: PF = 1 디폴트).

> [이미지 Untitled 59.png] [Tobe/A2500] SETUP > MEASUREMENT > Power — Phase/Total Power Calculation + **PF Sign "Signed", PF Value at No Load "PF = 1"** 통일 표기.

- 「HOME」 표시 통일: 피상전력 0(No Load)일 때 **"No Load"와 값을 같이 표시**(PF 0이면 "No Load"+0.000, PF 1이면 "No Load"+1.000).

> [이미지 Untitled 60.png] [Tobe/A2500] HOME > POWER > PF — A/B/C/Total에 **"No Load" + 1.000** 동시 표시(하이픈 없는 "No Load" 표기).

- (2024/01/25 추가) **최소계측 미만 소스 인가 시**에도 피상전력이 0으로 표시되므로 "No Load" 표시(소스 미인가와 동일 화면).
- (2024/06/04 추가, 2024/07/26 "-1.000" 추가) PF Value가 **반올림되어 "±1.000"으로 표시될 때 Angle은 None(공백)**:
  - PF Value와 Angle을 하나의 값처럼 표시하는 경우(장치 화면, AccuraSM/DM, PowerDX) 해당 — 순시·Min·Max 모두. 통신맵 값은 그대로(예: 0.9999 & Lag → 화면 1.000+공백, 통신 0.9999+Lag).
  - 회의 없이 담당자(조국춘·문부기) 협의 결정 → Policy-23-0018 음수 표기 협의 중 재검토 요청 → 기술영업부 협의로 유지 확정.

> [이미지 Untitled 61.png] [참고/A3700 User Guide Rev2.50] 역률 정의 — PF=P/S, 피상전력 0이면 연산 불가(1.0/0.0 표시 설정, 디폴트 1.0), 부호 제거 시 |P/S| + Fig 4.9 전력부호와 역률 위상각 Lead/Lag 4분면도(Q=0이면 Lead도 Lag도 아님).

> [이미지 image 12.png] [참고/조국춘, A2300 자료] PF sign 정리 — PF=|cosθ|(항상 양수) vs PF=cosθ(P의 부호) + 4분면도 + 표: S≠0/S=0별 PF값, **P=0일 때 Q>0이면 Lag·Q<0이면 Lead·Q=0이면 None**(빨간 박스).

- (2024/08/23 추가) **A3000F·A3300EN은 본 정책 미반영, 현재 구현 유지**:

> [이미지 image 13.png] [참고] 조국춘 이메일(2024-08-23) — "(A3000F·A3300EN 공통) 장치 구현에 통신매뉴얼을 맞추기로 함. 최종: **0: INVALID 및 NONE(피상·무효전력 0), 1: LEAD, 2: LAG**" + A3000F 통신매뉴얼 11070~11073 PF angle status 표.

> [이미지 image 14.png] [참고] 양준우 이메일(2024-08-19) — ModbusPoll 캡처와 함께 A3300EN angle status 실측 문의(0/1/2 응답 확인).

> [이미지 image 15.png] [참고] 조용화→문부기 전달(2024-08-19) — 개발자 전달 내용과 연구부 제품 설계 매뉴얼(한글/영문)의 Power Factor Angle Status 정의 차이 문의(0: None/유효하지 않음, 1: Lead, 2: Lag, 3: No load 혼재).

## 결정 배경

- No Load일 때 설정항목·값 표시가 장치마다 다름. 「PF at No-Load」 설정에 따라 HOME 표시: PF 0 Display는 "Invalid"+0.000, PF 1 Display는 숫자만 1.000 표시.

> [이미지 Untitled 62.png] [Asis/A2500] SETUP > Power — 항목명 "**PF at No-Load**", 값 "**PF 1 Display**"(A3700N·A7300과 상이).

> [이미지 Untitled 63.png] [Asis/A3700N·A7300] SETUP > Power — "**PF Value at No-Load**", "**PF = 1**" + Reactive Power Sign 항목.

- 표시 2가지 안 논의, **1안(No-Load+값 표시) 채택**:

> [이미지 Untitled 64.png] [1안/A2500] HOME > PF — "No-Load" + 1.000 동시 표시(채택안, 당시 하이픈 표기).

> [이미지 Untitled 65.png] [참고] 안창모 이메일(2023-04-26) — "기존 Invalid는 Lead/Lag와 별개로 부하 유무 구분으로 이해됨. 직관적으로 무부하임을 판별할 수 있어 좋은 표현. **Invalid를 No Load로 변경하는 것이 좋겠다**."

> [이미지 Untitled 66.png] [참고] 안창모 이메일(2023-04-24) — "No Load가 직관적이어서 좋을 듯".

  - 스마트솔루션팀(김태수): 부하유무가 같이 표현되는 현재 방식 선호(향후 PF 차트에서 Angle+Value 동시 표시 예정).
  - 2안(값만 표시) 검토 자료:

> [이미지 Untitled 67.png] [2안 검토] 조국춘 이메일(2023-04-25) — "S=0이면 위상각 상태를 No-Load로 표시하기보다 **표시하지 않는 것**이 매뉴얼 표기·정책상 맞음" + A3700N 통신매뉴얼 10135 Angle of PFa(0 None/1 Lead/2 Lag/3 유효하지 않음) 분석, Q=0이면 Lead/Lag 미표시.

> [이미지 Untitled 68.png] [2안 검토] 조국춘 전달(2023-04-25) — A3000F 통신매뉴얼은 "3: Invalid(피상전력 0)"가 0(유효하지 않음)으로 합쳐져 있음 확인 요청.

> [이미지 Untitled 69.png] [2안 검토] 조성운 의견(2023-04-26) — "Power phasor가 x-y축상에 존재하면 Display 표시 안 함": 0 undefined(S=0)→표시 안 함, 1 90-phase(P=0)→안 함, 2 in-phase(|P|=|S|)→안 함, 3 lead/4 lag만 표시.
